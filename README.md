# Trabalho-LP2-mercado
Este projeto foi desenvolvido em Java como parte da avaliação da atividade Linguagem de Programação 2. Ele modela um sistema de supermercado com base nos principais pilares da Programação Orientada a Objetos. Composição e Associações O projeto aplica a relação estrutural entre as classes Produto e Fornecedor. A classe abstrata Produto possui um atributo interno do tipo Fornecedor , o que significa que o produto não é apenas ligado ao fornecedor, ele o detém como parte de seus dados obrigatórios para ir às prateleiras. Abstração e HerançaFoi criada a superclasse Produto como abstrata, garantindo que não existam instâncias genéricas de produtos no sistema. A partir dela, desenhamos a relação com as classes filhas:ProdutoAlimenticio herda de Produto  e adiciona o campo de dias de validade.ProdutoEletronico herda de Produto  e gerencia os meses de garantia.
Em ambos, a palavra-chave super foi usada nos construtores para evitar reescrita e reaproveitar os atributos comuns da superclasse. Polimorfismo e SobrescritaO método calcularPrecoFinal() foi declarado como abstrato na classe Produto, obrigando suas subclasses a implementá-lo.
Utilizamos a anotação @Override nas classes filhas para definir regras de negócios distintas:Alimentício: Margem de lucro de 20%, recebendo 30% de desconto adicional no final caso vença em 5 dias ou menos.Eletrônico: Margem de lucro de 50%, com acréscimo de 15% de imposto no valor final caso possua garantia maior que 12 meses.O comportamento polimórfico se prova dentro de CarrinhoDeCompras. Ao criar um laço de repetição iterando sob uma lista genérica do tipo Produto, o Java determina em tempo de execução qual método calcularPrecoFinal() específico (do Alimento ou do Eletrônico) invocar para cada objeto lido, garantindo o cálculo seguro e flexível. Coleções de DadosO gerenciamento dos múltiplos itens foi realizado empregando a classe ArrayList do pacote java.util (através da interface genérica List<Produto>). Isso possibilita escalar o carrinho dinamicamente de acordo com as requisições de compra, garantindo fácil inserção e percurso (processarCompra()) ao invocar os resumos. Todo o encapsulamento foi preservado ao longo das classes utilizando modificadores de acesso (private), além de construtores estruturados e getters apropriados.
